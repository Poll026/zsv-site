### 1.2. Дизайн-система

- **Цвет:** графит/кремовый + золото `#C9A961` (есть переключатель тем `COLOR 1/2/3` сверху страницы — color2 «forest green», color3 «deep violet»)

- **Чередование DARK ↔ LIGHT секций**, тонкие переходные strip-полоски между блоками с `01/11`, `02/11`...

### 1.3. Визуальные эффекты (все из react-bits, конвертированы из React/OGL → vanilla Three.js)

| Эффект | Где | Описание |
|---|---|---|
| **LightRays** | hero (01) | Золотые лучи света сверху-центр, реагируют на курсор. Bright. Color `#F8E8C4` |
| **TextType (typewriter)** | блоки 02, 06 | Заголовки печатаются буква-за-буквой с золотым мигающим курсором при попадании в viewport |
| **Threads** | блок 03 | Деликатные диагональные нити в углах верх-лево/низ-право (mask) |
| **Particles + connections** | блок 05 | Крупные золотые звёзды-узлы, k-nearest neighbor граф, реагирует на курсор. Семантическая модель |
| **MagicRings** | блок 07 | Концентрические анимированные кольца, react на курсор |
| **LineWaves** | блок 09 | Очень слабый (10% от исходной интенсивности) глитч диагональных волн |
| **LetterGlitch** | футер | Сетка золотых символов меняется каждые 90ms, радиальная маска |
| **Snake-DNA line** | сквозь все блоки | Двойная золотая нить ДНК через геометрические центры всех 11 блоков (left↔right zigzag), пунктир бежит, узлы пульсируют |
| **Scroll thread** | левый край viewport | Вертикальная золотая нить-индикатор скролла, 11 точек по этапам, активная пульсирует, snippet `01/11` снизу |
| **Custom cursor** | весь сайт | Золотая точка + кольцо с инерцией, expand на hover-объектах |
| **GSAP scroll-reveals** | все блоки | Большие заголовки slide-up (y:80, 1.15s), подконтент fade-up (y:30, 0.85s) |
| **Magnetic buttons** | все .btn | CTA слегка тянутся за курсором (factor 0.25) |
| **Lenis smooth scroll** | вся страница | Премиум-инерционная прокрутка |

### 2.2. Шаги деплоя (примерный план — уточни с пользователем)

```
1. На VDS установить nginx (apt install -y nginx) — раньше его не было, был только OpenSSH+бот
2. Создать каталог /opt/zaharov/site/
3. Склонировать репо туда:
   sudo -u <admin-user> -H git clone https://github.com/Poll026/zsv-site.git /tmp/zsv-clone
   sudo cp -r /tmp/zsv-clone/ZSV/* /opt/zaharov/site/
   (или git submodule / или скрипт обновления — обсудить с пользователем)
4. Создать nginx-конфиг для zaharovsv.ru — server block слушает 80, root /opt/zaharov/site/, index index.html
5. ВАЖНО: открыть порты 80/443 в ufw — это требует явного согласования с пользователем (правило playbook)
   sudo ufw allow 'Nginx Full'  # или 80,443/tcp
6. DNS-настройка у регистратора: A-запись zaharovsv.ru → 85.209.154.225 (это пользователь делает в панели регистратора)
7. После DNS — установить SSL-сертификат через certbot:
   sudo apt install -y certbot python3-certbot-nginx
   sudo certbot --nginx -d zaharovsv.ru -d www.zaharovsv.ru
8. Проверить: открыть https://zaharovsv.ru, всё ли грузится (Three.js, GSAP, шрифты с Google CDN)
9. Настроить автоматическое обновление: cron или git pull при пуше — спросить у пользователя как удобнее
```

- ❌ Не открывать UFW-порты без явного согласия пользователя
- ❌ Не ломать существующий `/opt/leshik` (бот Лёшик там работает)
- ✅ Спросить про порты 80/443 явно: «открываю их под nginx, ок?»
- ✅ Обновления через `git pull` от **проектного** пользователя с флагом `-H`
